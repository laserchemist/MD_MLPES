📚 Framework Overview

What is this framework?
A complete, production-ready system for creating machine learning potential energy surfaces (ML-PES) from quantum chemistry calculations, with adaptive refinement and production molecular dynamics capabilities.

Key Capabilities

⚡ Direct Dynamics

Generate training data using PSI4 quantum chemistry calculations with automatic trajectory generation.

🤖 ML Training

Train accurate ML models with Coulomb matrix descriptors and Kernel Ridge Regression (<1 kcal/mol RMSE).

🔄 Adaptive Refinement

Two-phase diagnostic workflow with automatic error-based refinement (30-50% improvement per iteration).

🚀 Production MD

Run molecular dynamics 100-1000x faster than quantum chemistry using trained ML models.

📊 Visualization

Publication-quality plots including energy trajectories, temperature evolution, and 3D structures.

💾 Data Management

Multi-format trajectory I/O, metadata tracking, and computation database for easy organization.

💻 Code Examples

custom_training_data.py
#!/usr/bin/env python3
"""Generate training data with custom settings."""

from modules.direct_md import DirectMDConfig, run_direct_md
from modules.test_molecules import get_molecule
from modules.data_formats import save_trajectory
import numpy as np

# Get starting geometry
coords, symbols = get_molecule('formaldehyde_oxide')

# Run MD at multiple temperatures
temperatures = [50, 100, 150, 200, 250]
all_coords = []
all_energies = []
all_forces = []

for temp in temperatures:
    print(f"\n🔥 Running MD at {temp}K...")
    
    config = DirectMDConfig(
        method='B3LYP',
        basis='6-31G*',
        temperature=temp,
        n_steps=500,
        timestep=0.5,
        save_every=5
    )
    
    traj = run_direct_md(symbols, coords, config, use_ml_pes=False)
    
    all_coords.append(traj.coordinates)
    all_energies.append(traj.energies)
    all_forces.append(traj.forces)

# Combine all trajectories
combined_coords = np.vstack(all_coords)
combined_energies = np.concatenate(all_energies)
combined_forces = np.vstack(all_forces)

# Save
from modules.data_formats import TrajectoryData

combined_traj = TrajectoryData(
    symbols=symbols,
    coordinates=combined_coords,
    energies=combined_energies,
    forces=combined_forces,
    metadata={
        'temperatures': temperatures,
        'theory': {'method': 'B3LYP', 'basis': '6-31G*'}
    }
)

save_trajectory(combined_traj, 'custom_training_data.npz')
print(f"\n✅ Saved {len(combined_energies)} configurations!")

batch_md.py
#!/usr/bin/env python3
"""Run MD at multiple temperatures in batch."""

import subprocess

MODEL = "outputs/.../retrained_mlpes_model.pkl"
DATA = "outputs/.../augmented_training_data.npz"
TEMPS = [100, 200, 300, 400, 500]
STEPS = 50000

for temp in TEMPS:
    output = f"md_{temp}K.npz"
    
    print(f"\n🔥 Running MD at {temp}K...")
    
    cmd = [
        "python3", "simple_production_md.py",
        "--model", MODEL,
        "--training-data", DATA,
        "--temp", str(temp),
        "--steps", str(STEPS),
        "--output", output
    ]
    
    subprocess.run(cmd, check=True)
    
    # Visualize
    subprocess.run(["python3", "visualize_trajectory.py", output, "--all"])

print("\n✅ Temperature study complete!")
custom_analysis.py
#!/usr/bin/env python3
"""Custom trajectory analysis."""

from modules.data_formats import load_trajectory
import numpy as np
import matplotlib.pyplot as plt

# Load trajectory
traj = load_trajectory('production_md_300K.npz')

# Calculate bond distances
def calculate_bond(coords, i, j):
    return np.linalg.norm(coords[:, i, :] - coords[:, j, :], axis=1)

# C-O bond distance over time
co_distance = calculate_bond(traj.coordinates, 0, 1)

# Plot
plt.figure(figsize=(10, 6))
plt.plot(co_distance, linewidth=1)
plt.xlabel('Frame')
plt.ylabel('C-O Distance (Å)')
plt.title('C-O Bond Evolution')
plt.grid(True, alpha=0.3)
plt.savefig('co_bond_analysis.png', dpi=300)

print(f"Mean C-O distance: {co_distance.mean():.3f} Å")
print(f"Std: {co_distance.std():.3f} Å")

🔧 Troubleshooting

Symptoms:

Mean error: 3397 kcal/mol
All errors very similar

Diagnosis:

python3 diagnose_huge_errors.py --model model.pkl --training-data data.npz

Common Causes:

  1. Coordinate unit mismatch (Angstrom vs Bohr)
    • Fixed in two_phase_workflow.py v2.2
  2. Energy reference offset
    • Check if energies are relative vs absolute
  3. Theory level mismatch
    • Training: HF/STO-3G, Validation: B3LYP/6-31G*
    • Re-generate with consistent theory
Solution: Use latest two_phase_workflow.py (units fixed). Verify theory levels match.

Symptoms:

PSI4 error: SCF failed to converge
MD stuck at step 234

Solution:

# In PSI4 settings:
psi4.set_options({
    'maxiter': 500,          # Increase from 200
    'e_convergence': 1e-8,   # Tighter
    'd_convergence': 1e-8,
    'damping_percentage': 20  # Add damping
})

Symptoms:

Original RMSE: 5.2 kcal/mol
After refinement: 5.1 kcal/mol (only 2% improvement)

Possible Causes:

  1. Model converged (good sign!)
  2. Descriptor not expressive enough
  3. Not enough high-error points
  4. Systematic bias

Solutions:

  • Accept current quality if < 5 kcal/mol
  • Try different descriptor (SOAP, MBTR)
  • Use strategy [1] (add all points)
  • Consider different ML method (neural network)

Symptoms:

MemoryError: Unable to allocate array

Solutions:

  1. Reduce validation set size: 
    python3 two_phase_workflow.py --validate-max 25  # Instead of 50
  2. Reduce memory usage:
    • Smaller batches
    • Fewer hyperparameter combinations
    • Use simpler descriptors

Symptoms:

TypeError: TrajectoryData.__init__() missing 1 required positional argument: 'forces'

Solution:

You're using an old version of simple_production_md.py! Re-download the fixed version from the chat.

Or manually fix by adding forces saving in MD loop (see QUICK_FIX_FORCES.md)
